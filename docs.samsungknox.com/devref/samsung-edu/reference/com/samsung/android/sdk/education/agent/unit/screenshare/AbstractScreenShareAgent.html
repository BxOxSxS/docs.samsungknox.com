public void startScreenShare (ScreenShareTarget from, ScreenShareTarget to, T type, boolean isSessionEncrypted)

Starts screen share.
It can be 1:1 , 1:N and N:1 (Sender : Receiver).
A device selected as a sender captures its own screen and sends it to receiver(s) repeatedly.
Sending data is encoded by selected codec. (Jpeg, ICON, H.264 ...)
When a device selected as a receiver receives image data, the device decodes data and show the image.

If user set surface( android UI component ), draw image on the surface.
If user dose not set surface, we send bitmap image through ScreenShareListener. The user have to show bitmap image on the screen on yourself.
H.264, Streaming type supports surface only. And thumbnail type supports bitmap image only. Screen codec supports both of them.

Parameters
from Sets sender. Use static method of 'Target' class.
to Sets receiver. Use static method of 'Target' class.
type Sets screen sharing type.
isSessionEncrypted Sets if sender wants to encrypt session. This param is being considered only during H.264 streaming.
Throws
UnsupportedOperationException If this does not support this feature, throws UnsupportedOperationException
IllegalArgumentException If argument is not good, throw IllegalArgumentException.
For example, sender equals with receiver, no sender, no receiver or etc...
IllegalStateException While sending screen data using H.264 codec, can not start screen share again without stopping.
If try it, throw IllegalStateException.

public void startScreenShare (ScreenShareTarget from, ScreenShareTarget to, T type, boolean isUsingMediaProjection, boolean isSessionEncrypted)

Starts screen share.
It can be 1:1 , 1:N and N:1 (Sender : Receiver).
A device selected as a sender captures its own screen and sends it to receiver(s) repeatedly.
Sending data is encoded by selected codec. (Jpeg, ICON, H.264 ...)
When a device selected as a receiver receives image data, the device decodes data and show the image.

If user set surface( android UI component ), draw image on the surface.
If user dose not set surface, we send bitmap image through ScreenShareListener. The user have to show bitmap image on the screen on yourself.
H.264, Streaming type supports surface only. And thumbnail type supports bitmap image only. Screen codec supports both of them.

For H.264 streaming it is possible to choose Sender screen capture method.
Setting to true will force sender to use Android's Media Projection API to capture Screen.
Setting to false will force sender to use Knox SDK API to capture Screen.

To use this API with Media Projection it is required to call setMediaProjectionResult(int, Intent) before starting Screen Share.
Media Projection API cannot be used with SmartView feature together. Screen Share will be stopped when user turn on SmartView feature during H.264 Screen Share.

Parameters
from Sets sender. Use static method of 'Target' class.
to Sets receiver. Use static method of 'Target' class.
type Sets screen sharing type.
isUsingMediaProjection Sets if sender should use Media Projection API to capture screen. This param is being considered only during H.264 streaming
isSessionEncrypted Sets if sender wants to encrypt session. This param is being considered only during H.264 streaming.
Throws
UnsupportedOperationException If this does not support this feature, throws UnsupportedOperationException
IllegalArgumentException If argument is not good, throw IllegalArgumentException.
For example, sender equals with receiver, no sender, no receiver or etc...
IllegalStateException While sending screen data using H.264 codec, can not start screen share again without stopping.
If try it, throw IllegalStateException.

public void stopScreenShare (T type)

Stop screen share.
If this device is now sending its screen data to other(s), stop sending and notify to UI through ScreenShareListener. Then receiver(s) stop receiving also.
If this device is now receiving other device's screen data, stop receiving and notify to UI through ScreenShareListener. Sender stop sending screen data also.

Parameters
type Set screen sharing type.
Throws
UnsupportedOperationException If this does not support this feature, throws UnsupportedOperationException
