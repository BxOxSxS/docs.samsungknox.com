Manage profiles

The Profiles tab provides a centralized view of customer profiles and allows you to copy profiles to any of your fully or jointly managed customers. Currently, Knox Configure profiles are supported in the Knox MSP Portal. Profiles from other Knox services are planned for future releases.

Knox Configure profiles

The KNOX CONFIGURE PROFILES tab of the Profiles page displays a complete list of Knox Configure profiles, including profiles created in customers' Knox Configure consoles and profile copies created in the Knox MSP Portal.

The profiles table provides information on:

  • PROFILE NAME — The name of the profile. Click the name to open the Profile page in the customer's Knox Configure console that contains this specific profile.
  • CUSTOMER — The name of the customer's enterprise.
  • SERVICE — The Knox service that contains the profile.
  • PROFILE INFO — Knox version that the profile supports.
  • PROFILE TYPE — Knox Configure profile type (Setup or Dynamic).
  • TOTAL DEVICES — The total number of devices to which the profile has been assigned.

  • ORIGIN PROFILE — If the profile is a copy, indicates the original profile it was copied from. Click the name to open the Profile page in the customer's Knox Configure console that contains this specific profile.

    • If the profile is not a copy, a dash (-) is displayed.
    • If the origin profile was changed since the profile copy was made, is displayed. Hover over the icon for more details.
    • If the origin profile was deleted since the profile copy was made, is displayed and the link to the profile is removed. Hover over the icon for more details.
  • UPDATED — The date the profile was last modified, including any changes made by the customer on the customer's own console.

Edits to the origin profile aren't applied to copied profiles. You must first update the origin profile, then copy the changes across to the profile's copies. For more information, see Update a profile.

You can customize the columns displayed in the profiles table through the CUSTOMIZE TABLE menu. To access the menu, click at the top-right corner of the table.

You can also generate a CSV file report of all profile information by clicking DOWNLOAD PROFILE(S) AS CSV.

The report contains detailed information on:

  • PROFILE NAME — The name of the profile.
  • CUSTOMER — The name of the customer's enterprise.
  • SERVICE — The Knox service that contains the profile.
  • KNOX VERSION — Knox version that the profile supports.
  • PROFILE VERSION — The actively deployed version of the profile.
  • PROFILE TYPE — The Knox Configure profile type (Setup or Dynamic).
  • TOTAL DEVICES — The total number of devices to which the profile has been assigned.
  • ORIGIN PROFILE — If the profile is a copy, indicates the original profile it was copied from.
  • UPDATED TIME — The date when the profile was last updated.
  • ORIGIN PROFILE UPDATED — The date when the origin profile was last updated, either by the MSP or the customer.

Copying profiles

Existing Knox Configure profiles from fully managed customers can be copied and applied to other fully or jointly managed customers. You can quickly update profile information or create profiles for new customers by copying a profile.

Copying profiles reduces the risk of errors that often occur when manually duplicating settings across many profiles or recreating a profile from scratch. If you are managing a new customer that requires Knox Configure profiles similar to that of an existing customer, you can copy these existing profiles to your new customer. After copying, you can continue to make customer-specific profile edits on the customer's console.

NOTE

  • Copying a profile won't duplicate any Routine Applets assigned to it. You must manually manage routines on the customer's console.
  • Copying a profile also won't copy any Product Information Screen settings configured during profile creation. You must enter this information manually.

The COPYABLE PROFILES tab displays a list of profiles that you can copy. See the section Add a copyable profile for more details.

The list of copyable profiles contains the following information:

  • PROFILE NAME — The name of the profile that can be copied. This is also referred to as origin profile in the rest of the console. Click the name to open the Profile page in the customer's Knox Configure console that contains this specific profile.
  • PROFILE INFO — Knox version and profile version information (Knox Configure only).
  • PROFILE TYPE — The Knox Configure profile type (Setup or Dynamic).
  • CUSTOMER — The name of the customer's enterprise.
  • ORIGINAL PROFILE UPDATED — The date when the original profile was last updated.

You can customize the columns displayed in the copyable profiles table through the CUSTOMIZE TABLE menu. To access the menu, click at the top-right corner of the table.

You can also generate a report for specific profiles as a CSV file. Select the profiles you wish to download, then click ACTIONS > Download selected profile(s) as CSV.

The report contains detailed information on:

  • PROFILE — The name of the profile which can be copied. This is also referred to as origin profile in the rest of the console.
  • CUSTOMER — The name of the customer's enterprise.
  • SERVICE — The Knox service that contains the profile.
  • KNOX VERSION — Knox version that the profile supports (Knox Configure only).
  • PROFILE VERSION — The actively deployed version of the profile (Knox Configure only).
  • PROFILE TYPE — The Knox Configure profile type (Setup or Dynamic).
  • TOTAL DEVICES — The total number of devices to which the profile has been assigned.
  • ORIGIN PROFILE — If the profile is a copy, indicates the original profile it was copied from.
  • UPDATED TIME — The date when the profile was last updated.
  • ORIGIN PROFILE UPDATED — The date when the origin profile was last updated.

Add a copyable profile

In order to copy a profile to customers, you must first make it available for copying. Only profiles from fully managed customers can be made available for copying.

When you make a profile available for copying, all customizations, including customer support information (customer email, phone number, company name), are carried over. With this in mind, it's highly effective if you first create a template profile with placeholder customer support information. This enables you to copy the profile over to multiple customers without the risk of accidentally sharing sensitive information. Customer-specific edits can be made after the profile is copied.

To add a copyable profile:

  1. On the COPYABLE PROFILES tab, click ADD COPYABLE PROFILES.

  2. Select the profiles you wish to copy. Optionally, filter the list of profiles by entering a fully managed customer name, customer ID, or profile name in the search bar.

  3. Click NEXT. The selected profiles are added in the copyable profiles table.

Delete a profile from the copyable profiles table

Removing profiles from the table of copyable profiles doesn't delete the original customer profile or any profile copies created from it.

To remove a profile from the copyable profiles table:

  1. Select the profiles you wish to delete.
  2. Click ACTIONS > Remove copyable profile(s) from table.
  3. Click REMOVE to confirm the deletion. The profile is removed from the list of copyable profiles.

Copy profile to customer

One profile can be copied to many customers at once. This action is useful when you need to quickly push an update to multiple customers at once. In addition, multiple origin profiles can be copied to one or many customers, enabling you to quickly set up Knox Configure profiles for new customers. For more information, see Copy multiple profiles to multiple customers.

Each customer can only have one profile with a given name. If you copy a profile to a customer that has a profile with the same name, the portal warns you. To create a new profile copy, assign the copy a different PROFILE NAME. For more information, see Update a profile.

NOTE — Overwriting a profile replaces all of its content, with a few exceptions. Customer support information (customer email, phone number, company name) and animations won't be changed. However, any customizations or customer-specific edits to other sections will be removed. For more information, see Update a profile.

To copy a profile to a customer:

  1. On the COPYABLE PROFILES tab, select the profiles you wish to copy.

  2. Click ACTIONS > Copy to customer(s).

  3. Select one or more customers you wish to copy the origin profile(s) to. Up to 10 customers can be selected at a time. To remove an origin profile, click X under Selected profiles. To remove a customer from the list of target customers, click REMOVE under Selected customers.
  4. Click NEXT.

  5. (Optional) Customize the name of the newly copied profile(s).

  6. (Optional) Before continuing, if you wish to prevent a profile from being copied, select the checkbox next to the profile's name.
  7. Click CREATE PROFILES to finish copying the profiles. The copied profiles appear in the KNOX CONFIGURE PROFILES table.

Copy multiple profiles to multiple customers

Multiple profiles can be copied to multiple customers at the same time.

Each customer can only have one profile with any given name. If you copy a profile to a customer that already has a profile with the same name, the portal warns you. To create a new profile copy, assign the copy a different PROFILE NAME.

NOTE — Overwriting a profile replaces its content, with a few exceptions. For more information, see Update a profile.

To copy multiple profiles to multiple customers:

  1. On the COPYABLE PROFILES tab, select the profiles you wish to copy.
  2. Click ACTIONS > Copy to customer(s).

  3. Select the customers you wish to copy the origin profiles to. Up to 10 customers can be selected at a time. To remove an origin profile, click X under Selected profiles. To remove a customer from the list of target customers, click REMOVE under Selected customers.

  4. (Optional) Customize the name of the newly copied profiles.

  5. (Optional) Before continuing, if you wish to prevent a profile from being copied, select the checkbox next to the profile's name.
  6. Click CREATE PROFILES to finish copying the profiles. The newly copied profiles appear in the KNOX CONFIGURE PROFILES table.

Update a profile

You can update a profile copy by overwriting it with its origin profile. Origin profiles can be identified in the ORIGIN PROFILE column of the KNOX CONFIGURE PROFILES table.

Updating a profile replaces the profile's content with that of the origin profile, with a few exceptions:

  • Customer support information (customer email, phone number, company name) won't be overwritten. This is by design, as to avoid overwriting individual customer customizations.
  • As Routine Applets and Product Information Screen settings aren't copied over when a profile copy is made, they can't be updated through profile updating. For more information, see Copying profiles.
  • As of the 22.11 release, animations aren't supported by profile updating. When a profile is copied, the copy will inherit any animations from its origin profile. However, updates to animation files aren't applied to profile copies; they will keep the old animations, even after the origin profile has been updated. But, if the animations are removed from the origin profile during profile updating, they will also be removed from any profile copies.
  • As of the 22.11 release, when an update is pushed to a Dynamic profile containing non-APK files (such as images and ringtones), devices to which the profile is assigned will always redownload these files regardless of whether they have been updated or not. This doesn't apply to animations. Furthermore, the update will occur regardless of whether the device was originally enrolled in Knox Configure through a Setup or Dynamic mode profile.

Only profile copies created on the Knox MSP Portal can be overwritten. In addition, you can only overwrite a profile copy with its origin profile.

If you attempt to overwrite a profile copy with a profile that isn't its origin profile, or if the profile you are attempting to overwrite is not a copy, the portal prevents the copy and shows an error.

To update a profile:

  1. Identify the origin profile of the profile copies you wish to update. To help with this, you can refer to the ORIGIN PROFILE column of the regular profile list on the KNOX CONFIGURE PROFILES tab. Make your desired changes to the origin profile of the profile copy you wish to update.

  2. On the COPYABLE PROFILES tab, select the origin profile of the profile copies you wish to update.
  3. Click ACTIONS > Copy to customer(s).
  4. Select the customers to which you wish to update the profiles. Up to 10 customers can be selected at a time. To remove an origin profile, click ✕ next to it under Selected profiles. To remove a target customer, click REMOVE under Selected customers.

  5. Click NEXT. The portal shows the profiles selected for updating.

  6. (Optional) Before continuing, if you wish to prevent a profile from being updated, select the checkbox next to the profile's name.
    7. IMPORTANT — A profile can be overwritten even if its name is different from the name of its origin profile. However, if you change the PROFILE NAME of a profile copy during name customization, the profile will not be updated. Instead, a new profile copy will be created. Only the profiles that display an overwrite warning will be updated.
  7. Click CREATE PROFILES to finish updating the profile.
