Knox MSP Program 23.12 release notes

Last updated December 6th, 2023

MSP sub-admin permissions

You can now limit how MSP sub-admins can manage customer portals with role-based permissions.

When creating a role, you can choose from a set of permissions to allow or deny sub-admins access to a wide variety of customer portal features.

Email preference controls

Previously, you were unable to configure email preferences for customer Knox cloud services, which meant that you would receive email notifications from all managed customer services.

With this release, you can now opt in and out of emails from customer Knox cloud services that you manage.

Notification emails sent to MSP admins have also been rewritten to improve their formatting, provide clear calls to action, and create consistency with other Knox services. In addition, the customer name is now included in the subject line.

Customer device alerts

When viewing customer details on the Customers page, you can now see the total number of devices that need attention to help quickly identify and fix issues. Device counts are divided by service to allow you to easily identify issues.

Deletion of Knox Suite licenses

Previously, a Knox Suite license could only be deleted if it wasn’t active nor assigned to any devices.

With this release, you can now delete a Knox Suite license that’s actively used by Knox Platform for Enterprise, Knox Capture, or Knox Authentication Manager.

Additional Knox Suite license details

When viewing the details of a Knox Suite license on the Licenses page, you can now see the consumed seat count for each service. If the license is shared between multiple customers, you can also see the consumed seat count for each customer.

Creation and renewal of Knox Platform for Enterprise licenses

You can now generate and renew Knox Platform for Enterprise licenses for fully or jointly managed customers from the Licenses page of the customer’s Knox Admin Portal.

Note that since these licenses are generated on the customer’s portal, they can’t be shared with other customers.

Security update for PII storage and search

This release delivers a significant update to Knox cloud infrastructure that further hardens how fields with personally-identifiable information (PII) are stored. This update impacts how you can search for PII across all Knox services.

For the technical details about this update, as well as an up-to-date breakdown of how this affects the Knox MSP Portal, see Security notice regarding storage and search behavior for PII.
