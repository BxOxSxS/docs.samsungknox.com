Knox STIG API table (Android 10)

About STIG

Any mobile device used for the US military must meet a minimum set of STIG requirements. This ensures the maximum protection for sensitive military data and improves the security of Department of Defense (DoD) information systems. Knox meets the STIG requirements when appropriate APIs are applied to lock down the device. These APIs are described in the table below.


For a full list of STIG documentation including past releases see the Knox certifications page.


Knox STIG API Table (Knox 3.x / Android 10)

This guide is an explicit list of the APIs required to reach STIG compliance on a Samsung Knox device. It is addendum to the official STIG configuration tables to help remove any ambiguous interpretation of MDM agnostic language used. The intended audience is either MDM developers responsible for implementing configuration controls, or Administrators (MDM) wishing to verify that an MDM has used the correct API.


Terms

  • Vendor - the device vendor. AE: Android Enterprise, KPE: Knox Platform for Enterprise
  • Group - the device feature category
  • Rule - the policy used to control the feature
  • Class - the class in the Knox SDK used to control the feature
  • Method - the API in the Knox SDK used to control the feature
  • COBO - Configuration of a device for work use only, with a single space for work applications and data. Personal applications and data are prohibited.
  • COPE - An enterprise-owned device for business and personal use. A KPE Workspace/Work profile is configured to separate work applications and data from personal applications and data.

Acronyms

  • AE - Android Enterprise
  • AO - Authorizing Official
  • API - Application Programing Interface
  • CC - Common Criteria
  • COBO - Corporate Owned Business Only
  • COPE - Corporate Owned Personally Enabled
  • DA - Device Admin
  • DeX - Desktop Experience
  • DISA - Defense Information Systems Agency
  • DO - Device Owner
  • DoD - Department of Defense
  • DPC - Device Policy Controller
  • HID - Human Interface Device
  • KPE - Knox Platform for Enterprise
  • KPE(AE) - An Android Enterprise deployment using KPE capability
  • KPE(Legacy) - An Android Legacy deployment using KPE capability
  • MDM - Mobile Device Management
  • PO - Profile Owner
  • STIG - Security Technical Implementation Guide
  • USB - Universal Serial Bus
  • VPN - Virtual Private Network

Deployment Terms

  • Android Enterprise - Managed by a DPC in DO/PO mode
  • Android Legacy - Managed by a DPC in DA mode
  • Fully managed device - Android Enterprise managed device
  • Legacy managed device - Android Legacy managed device
  • Workspace - KPE legacy workspace
  • Work profile - KPE work profile (replaced legacy workspace)
