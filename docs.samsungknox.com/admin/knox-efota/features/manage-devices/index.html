Manage devices

Last updated August 5th, 2025

This section describes how to use tools in the Devices tab to manage devices in Knox E-FOTA. View device information, edit devices, delete devices, and label devices in this tab.

View device information

  1. Sign in to the Knox Admin Portal. In the left sidebar, click Knox E-FOTA.

  2. Go to Devices > DEVICES.

  3. (Optional) Enter a device ID in the Search field to search for a device.

  4. Review the list of devices.

To generate a list of devices, click Actions > Download devices as CSV. If no devices are selected, all devices are included in the CSV file.

Each device has the following details:

  • Device ID — Unique ID associated with the device.
  • Model — Device model.
  • Firmware — Firmware version installed on the device.
  • OS version — Android OS version installed on the device.
  • Security patch — Security patch of the firmware on the device.
  • Operation status — Describes the status of the device licensing and campaign sync.
  • Client status — Describes the status of an assigned campaign. If the client encountered an error, the client error is also shown. Learn more about Knox E-FOTA client errors.
  • Campaign — Campaign the device is assigned to (if any).
  • Gradual campaign rollout start — Whether gradual campaign rollout has reached this device.
  • Update — Status of device updates.
  • License — License key that the device is assigned to.
  • License ends — Date when the license expires.
  • Tags — Labels associated with the device (if any). There are two types:
    • System tags — Labels that Knox E-FOTA automatically assigns to devices. You can’t delete these. For example, DUMMY, which indicates that the device has a dummy binary installed through a test campaign.
    • Custom tags — Custom labels you can assign to devices.
  • Source — Method by which the device was added, such as through an EMM, reseller upload, or CSV file upload.
  • EMM group — Device group in the EMM that the device belongs to (if any).
  • EMM instance — EMM instance this device belongs to if it comes from an EMM sync.
  • EMM status — Status of the EMM instance this device belongs to if it comes from an EMM sync.
  • Auto-Enrollment — Indicates whether or not auto-enrollment with out-of-box experience (OOBE) is enabled.
  • Sales Code — Sales code for the device.
  • CSC — Country Specific Code for the device.
  • Modified — Date when the device was last modified. For example:
    • The device was added.
    • The device’s status changed.
    • The campaign changed.
    • The license changed.
    • The source changed.
    • The EMM group changed.
  • Last Seen — Date and time when the Knox E-FOTA server last pinged the device.
  • Order Number — Order number the device was purchased under.

Device statuses

This section describes the different statuses devices can have and what they mean:

  • Operation status: Describes the status of the device licensing and campaign sync.

  • Campaign status: Describes the status of an assigned campaign.

  • Client status: Describes the status of the Knox E-FOTA client app on the device, along with the associated status code.

  • Update status: Describes the status of a firmware update, information that is now also included in the Device Update Status dashboard and Campaign Graph.

For details about a status, hover over the status to display a tooltip description.

Client status

Error

These client statuses indicate that Knox E-FOTA encountered an error on the device. See the exact client error code for specific details about your error.

Client status Description
Failed to download Firmware failed to download, possibly due to network issues.
Failed to enroll Device couldn't enroll to Knox E-FOTA, check that your license is valid.
Failed to install Firmware installation failed. This could be because the device is rooted, or there is an issue with the firmware installation process.
Campaign error Campaign couldn't be read and applied.
Invalid firmware version Assigned target firmware is invalid

In-progress of an operation

These client statuses indicate that Knox E-FOTA is running or preparing to run an operation. See the exact client error code for specific details about your error.

Client status Description
Awaiting download Knox E-FOTA is waiting for firmware download conditions to be met. This can be due to firmware download timeframe, download network restrictions, or insufficient device storage.
Awaiting install Firmware has successfully downloaded, and Knox E-FOTA is waiting for installation conditions to be met. This can be due to firmware install timeframe, charging and battery restrictions, or insufficient device storage.
In-progress (downloading) Firmware is currently downloading.
In-progress (installing) Firmware is currently installing.
Reverting Dummy firmware binary was installed while the device was on a test campaign, and the device is currently reverting to a normal firmware binary.

Operation completed

These client statuses indicate that Knox E-FOTA has completed an operation, or was unable to begin an operation.

Client status Description
Higher firmware version The device's current firmware version is already higher than the firmware assigned to it in the campaign.
Locked firmware version Campaign is active and the device is locked to its current firmware version.
Updated Device update has completed.

Not started

These client statuses indicate that Knox E-FOTA hasn’t begun to actively manage firmware on your devices.

Client status Description
Enrolled Device is enrolled to Knox E-FOTA, but has no active campaign assigned.
Not enrolled Device is not enrolled. Either client is not installed or license is invalid.

Perform common operations on devices

You can perform the following operations on one or more devices:

  • Assign campaign
  • Create campaign
  • Replace license
  • Manage device tags
  • Unassign campaign
  • Unenroll devices
  • Delete devices
  • Force client refresh
  • Download devices as CSV
  • View device log
  • View the device deletion log

To perform a common operation on one or more devices:

  1. Sign in to the Knox Admin Portal. In the left sidebar, click Knox E-FOTA.
  2. Go to Devices.
  3. Select the checkboxes corresponding to the relevant devices.
  4. Click Actions and select the operation you want to perform.

Additionally, see Add devices to know more about adding devices.

View a device log

The device log shows a list of events that occurred on a device. For each event, the resulting device status is shown along with the timestamp.

  1. Sign in to the Knox Admin Portal. In the left sidebar, click Knox E-FOTA.
  2. Go to Devices.
  3. Click the device ID of the device you want to view.
  4. In the Device details popup, click View device log.

Tag a device

Tagging is useful if you want to label devices and easily search for them in the device list. For example, you can label devices by their business function or by the location of their device users. This section describes how to tag a single device. If you want to tag multiple devices at once with the same tags, see Perform common operations on devices.

  1. Sign in to the Knox Admin Portal. In the left sidebar, click Knox E-FOTA.
  2. Go to Devices.
  3. Click the device ID of the device you want to tag.
  4. In the Device details dialog, under the Tags field, enter the labels to associate with the device. For example: sales.
  5. Click Save.

Install the Knox E-FOTA client app

Your enterprise devices need to have the Knox E-FOTA client app installed before they can be managed by Knox E-FOTA.

At the bottom of the Devices page, click DIRECT DOWNLOAD to find the APK link to share with your device users. You can also find a QR code here, which you can share with your device users so they can scan it to download the Knox E-FOTA client app.

For more ways to install the Knox E-FOTA client app, see Install and launch the Knox E-FOTA client.
