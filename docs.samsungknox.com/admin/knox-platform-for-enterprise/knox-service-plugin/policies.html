The URL to the download server where the client can download client certificates in PKCS#12 format. The expected format is: http:///path/to/cerv/directory

MTU Size

Default packet MTU for the tunnels. The minimum value is 576, and the maximum value is 1500. Recommended value is 1280, which should work in most mobile networks.

Differentiated Services

Differentiated Services. This value is used to mark packets in order to enable Quality of Service. The minimum value is 0, and the maximum value is 63. To not use the feature, set it to 0.

DTLS Tunnel

Whether the DTLS (realtime sensitive data) tunnel should be used for the application data, or the application data instead should be sent over the TLS tunnel. The recommendation is, if possible, to use DTLS.

TCP Alive Frequency

The frequency (in seconds) of the client sending TCP Keep-alive messages to the server. This parameter has two uses: firstly, by continuously sending traffic it attempts to prevent networks from flushing the TCP/TLS tunnel, and secondly, as replies are expected from the server for each message, the client can ensure that the connection is still up. The minimum value is 1, and the maximum values is 18000. Note that the timeout value affects the power consumption of the device so this should be chosen with care. The recommended value is 1200 seconds (20 minutes), which should work in most mobile networks.

DTLS Inactivity Timeout

The allowed amount of time (in seconds) of silence in the DTLS tunnel before the client assumes that the tunnel is no longer used and closes it. The tunnel will automatically be re-established when needed. This parameter is used to prevent the UDP/DTLS tunnel from being flushed by the networks without the client and server knowing about it, by instead having a controlled shutdown of the tunnel before a flush occurs. The minimum value is 1, and the maximum value is 300. Recommended value is 30 seconds, which should work in most mobile networks.

USB Tethering

A group of policies to control USB tethering over VPN; feature will work only if the below conditions are met a. IT admin has to allow the feature from UEM console for device to get the tethering feature; b. User has to manually allow "USB tethering feature"; c. Laptop/Tablet being connected has to be previously allowlisted by IT admin.

Enable USB Tethering over VPN

Use this control to enable USB Tethering over VPN. If this control is not enabled, any changes you make to USB Tethering over VPN settings are ignored.
