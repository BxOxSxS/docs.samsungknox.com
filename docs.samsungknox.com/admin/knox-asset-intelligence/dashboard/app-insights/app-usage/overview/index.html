App usage overview

Last updated March 20th, 2025

To help you analyze how apps are used in your organization, Knox Asset Intelligence provides several insights related to which apps are in use, how long they’re in use, and the resources consumed by each app.

Available app usage insights

The following app-usage related insights are available in Knox Asset Intelligence:

  • App type
  • Total, Background, and Foreground app usage
  • Apps with highest battery consumption
  • Apps with highest network usage
  • Apps with highest RAM usage

View app usage insights

There are two ways to view app usage related insights: on the main Dashboard, or on the dedicated Apps sub-dashboard.

Main Dashboard

The main Dashboard is where you can view all Knox Asset Intelligence insights. To view app-usage related insights, you must first select these insights in the Dashboard Settings > CUSTOMIZE tab. See Configure dashboard settings for more details.

Dedicated Apps sub-dashboard

The Apps sub-dashboard groups all app-related dashboard insight, including insights related to app usage, into a single page. This page doesn’t require you to select each insight in Dashboard Settings. To access this page, click Apps under Dashboard in the left navigation pane.

Create app usage reports

To help you analyze your fleet’s app usage in more detail, you can generate an app usage report with customized data fields suited to your needs. To do this:

  1. Go to the expanded view page for that insight, then in the APPLICATION LIST table, select up to 10 apps to view in the report.
  2. Click CREATE APP USAGE REPORT.
  3. In the pop-up dialog, select the reporting period, either Yesterday, or Last 7 days.
  4. In the Data Type field, select the type of data you want to include in your usage report, then click DOWNLOAD USAGE REPORT. The report is sent to your email once ready.

The CREATE APP USAGE REPORT button is not available for the Apps with highest RAM usage and App type insights.

Alternatively, you can generate an app usage report from the Devices page. To do so:

  1. Go to the Devices page, then click a DEVICE ID to open the expanded device details panel.
  2. In the APPS field, click CREATE APP USAGE REPORT, then follow the same procedure above to create your report.

Use app usage reports

Once the app usage report is downloaded, you’ll have access to the following information, depending on the parameters selected:

  • IMEI: The device IMEI (International Mobile Equipment Identity), a unique 15-digit number assigned to a mobile device, used by network operators to identify and track devices on their network.
  • Serial number: A unique identifier used for inventory management, warranty tracking, and device identification
  • Device group: Name of the group this device belongs to.
  • Device model: The device’s model number.
  • App name: The name of the app that reported usage.
  • Package: The unique identifier that represents the app’s package in the Android operating system
  • App version: The version of the app that reported usage.
  • Date: The date of reported usage.
  • Total run time (minutes): The total combined foreground and background app run time.
  • Total foreground run time (minutes): The total time the app was running in the foreground.
  • Total background run time (minutes): The total time the app was running in the background.
  • Total battery consumption (percent): The total battery % consumed during the entire app run time.
  • Normalized battery consumption (percent): Total battery consumption % per hour of app run time.
  • Total network data usage (MB): The total combined mobile and Wi-Fi data usage during the app run time.
  • Total mobile data usage (MB): The total mobile (cellular) data usage during the app run time.
  • Total Wi-Fi data usage (MB): The total Wi-Fi data usage during the app run time.
  • Total times launched: The number of times the app launched during the reporting period.
  • Total screen time (minutes): The total time the app was on screen during the reporting period.
  • Screen time per app launch (normalized)(seconds): The total time the app was on screen for per app launch.
  • Average RAM usage (percent): The average RAM consumption for that app, over the reporting period.

About zero and null fields

There are times where the app usage report may show a zero (0) or null (empty) value in one column, despite showing a value in another corresponding column. For example, the app usage report may show an empty Average RAM usage value, but Total run time contains data.

For some fields, 0 indicates that the calculated value is too small, whereas an empty field can be caused by several factors:

  • The selected data in the report was not collected due to the data upload option being disabled in Dashboard Settings.
  • The selected data was not yet collected from the device, based on its own data collection policy.
  • The selected data was not yet uploaded to the server, based on its own data upload policy.
  • The selected data was not yet uploaded to the server due to network issues.
  • The selected device was using the application while charging. For some insights like App run time and Battery usage, data cannot be collected while the device is in a charging state.
