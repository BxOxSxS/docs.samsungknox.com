Knox E-FOTA One release notes - December 9, 2020

Knox E-FOTA One

Group hierarchy information for devices synced from SOTI

In SOTI, device groups can be arranged in hierarchies. That is, a device can belong to a group, and that group can in turn have a parent group. Previously, when adding a device from SOTI to Knox E-FOTA One, the IT admin only received information about the device group that device directly belonged to. Starting this release, the Knox E-FOTA One admin portal shows the hierarchical path of the devices so that IT admins can determine the hierarchical order of those devices in SOTI.

Each device’s full hierarchical path is shown in the Path field when they do the following:

  • Add or update the devices groups from SOTI (Update device groups on E-FOTA screen)
  • Visit the EMM groups tab.

Support for multi-CSC devices

To aid IT admins in selecting the firmware versions that are appropriate for their devices, the CSC (Consumer Software Customization) info of each device is now available:

  • When viewing devices in the Devices tab, the IT admin can customize the table columns to include each device’s CSC. When the IT admin exports the device list to a CSV file, the CSC info is also included.
  • When viewing a device’s Device details, the IT admin can now see that device’s CSC.
  • When assigning target devices to a campaign, the IT admin fills in the Assign Devices and Firmware table in the campaign creation screen. Each row of that table represents a device group (a model/sales-code pair). The CSC of each device group is now pre-populated next to the sales code. For device groups that have multiple possible CSCs, a dropdown-menu is displayed instead, so that the IT admin can select the CSC for that group.
  • When the IT admin moves on to select target firmware for a device group, they only see target firmware versions that are compatible with that device group given its model, sales code, and CSC.
  • When creating, editing, and viewing a campaign, the IT admin can now see the CSCs of devices in that campaign.

  • 500 error page

Client agent version in exported device info

When an IT admin downloads the device list as a CSV file, the CSV file will now contain the version of the Knox E-FOTA One app installed on each device. This allows the IT admin to get a better insight into the status of their devices.

Deploying a firmware version based on a device’s current firmware type

This release note applies only to enterprises that purchased the Software Customization Services license and have custom firmware registered in Knox E-FOTA One.

When the IT admin creates a campaign and selects firmware to deploy, the system behaves as follows:

If

Then

The IT admin selects Latest firmware for a particular group of devices*.

The latest firmware target is automatically defined as B2C or custom, depending on the current firmware type of that group of devices.

The IT admin selects Select from firmware list for a particular group of devices*, and a custom firmware version is currently installed on that group.

In the Select the target firmware screen that displays, only the Custom firmware tab is shown. This option prevents the IT admin from erroneously selecting normal firmware.

The IT admin selects Select from firmware list for a particular group of devices*, and a normal firmware version is currently installed on that group.

In the Select the target firmware screen that displays, only normal firmware versions are shown. This option prevents the IT admin from erroneously selecting custom firmware.

* A group of devices is defined as a combination of model, sales code, and CSC. It refers to a row in the Assign Devices and Firmware table shown in the campaign creation screen.

Forcing a client refresh

IT admins can now push a client refresh to devices they select in the Devices tab. In the Actions menu, Force client refresh has been added. This new menu item is only enabled if all selected devices have the client app installed.

When an IT admin selects this option, the following operations are performed:

  1. The current campaign info is deleted.
  2. The Knox E-FOTA One app is updated to the latest version.
  3. The latest campaign info is synced, and the campaign is restarted.

This operation is recorded in the device log and the Activity Log.

NOTE—Items to consider:
  1. This feature only works on devices with client version 1.1.16 or later.
  2. A push notification may take a maximum of 2 weeks, according to Google’s notification policy.
  3. This new Force client refresh button replaces the Push campaign button.
