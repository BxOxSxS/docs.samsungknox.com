Device management

Last updated April 9th, 2025

Devices

You can access the Devices page to view and manage your devices. Click on a device ID to open the DEVICE DETAILS panel.

Depending on a device’s current status and payment history, different actions are available in the ACTIONS menu; you can accept or reject devices, start and stop overdue payment notifications, send device payment reminders, lock and unlock devices, and complete subscription payments.

To view device information:

  1. On the Knox Guard navigation pane, select Devices. The Devices page opens and lists all available devices.

  2. You can search for a device by entering its IMEI or Approval ID into the search bar. The Approval ID search string can include the following special characters: @, :, $, /, *, %, ^, &, \ (, ), +, ?, {}, [].

  3. Select the information to view by clicking the ellipsis (…) on the far right of the table, and selecting the required columns. You can select a maximum of eight columns.

    Column Description
    Device ID Device identifier used in the Knox Guard console.
    Model Device model. You can filter the column to view specific device models.
    Status Current status of the device.

    Available statuses include: Activating, Active, Complete, Completing, Exchanging, Locked, Locking, Offline locked, Pending, Rejected, Reminder On, Resetting, Starting reminder, Stopping reminder, and Unlocking.

    For more information on a device’s status flow once added to the console, see Knox Guard status flow.
    Approval ID Billing approval ID used with a financial institution’s unique billing system to approve a loan payment.
    Enrollment notice Indicates whether an enrollment notice is enabled for the device.
    Offline lock Indicates whether the offline lock policy is enabled for the device.
    Auto-lock Indicates whether auto-lock is enabled for the device.
    Service ends Date after which the device is no longer supported by Knox Guard under the terms of its license.
    Carrier Device carriers that support the device. Optionally, filter the column to display specific carriers.
    Uploaded Upload status of the device.
    Accepted Indicates whether the device is accepted.
    Activated Indicates whether the device is activated.
    App version Current version of apps installed using the Application installation policy. Only available to devices with apps registered (uploaded) from the Knox Guard console.

    To enable Application installation, please contact your Knox Guard admin.
    Android version Current Android version of the device.
    Agent version Current agent version installed on the device.
    Modified Date when a device action was last performed (for example, Update offline lock).
    Firmware version Current firmware version of the device.
    Block factory reset Indicates whether factory reset is blocked for the device.
    App installation Indicates whether the Application installation policy is enabled.

    This feature is available only by request. To enable Application installation, please contact your Knox Guard admin.
    Latest firmware Latest available firmware version.
    Firmware update control Whether firmware update control settings are enabled for the device.
    App blocklist Whether the App blocklist policy is enabled.
    Last seen Date the device was last detected by the Knox Guard server.

    This feature is only available by request. To enable Last seen, please contact your Knox Guard administrator.
    Relock timestamp Time before the relock timestamp appears for the device. This column can only be selected by PAYG tenants.

  4. You can optionally download the device table as a CSV file.

  • If a device’s Status is Locked or Unlocking, a PIN button is shown to right of the status to indicate that a PIN must be entered on the user’s device to stop reminders or unlock the device.
  • If a device’s Status is Reminder On or Stopping reminder, no PIN button is shown.

Device details

Each device’s IMEI or SN appears as a link that you can select to view device details, such as Status/Last seen, SUMMARY, RESELLER, LICENSE, POLICIES, and NOTES.

You can query and retrieve the secondary IMEI (IMEI2) of dual-SIM devices using the Search function. IMEI2 numbers can also be exported through the ACTIONS menu by downloading the device table as a CSV.
