Download devices as CSV

Last updated November 19th, 2025

A sub-admin needs the View activity log permission to view and download the activity log. For more information, see Manage admins and roles.

Download all devices as CSV

To download device information as a CSV file:

  1. Navigate to the Devices page.

  2. Click ACTIONS > Download devices as CSV. If the number of devices exceeds 10,000, the file is generated asynchronously and becomes available in the Activity log when it’s ready for download.

    The CSV file will display the following device information:

    Column Description
    IMEI Device IMEI number.
    IMEI2 Device's secondary IMEI number, if applicable.
    SN Device serial number.
    MODEL Device model. You can filter the column to view specific device models.
    STATUS

    Current status of the device.

    Available statuses include: Activating, Active, Complete, Completing, Exchanging, Locked, Locking, Offline locked, Pending, Rejected, Reminder On, Resetting, Starting reminder, Stopping reminder, and Unlocking.

    For more information on a device's status flow once added to the console, see Knox Guard status flow.
    APP VERSION Current version of apps installed using the Application installation policy. Only available to devices with apps registered (uploaded) from the Knox Guard console.
    APPROVAL ID Billing approval ID used with a financial institution's unique billing system to approve a loan payment.
    LAST SEEN

    Date the device was last detected by the Knox Guard server.

    This feature is only available by request. To enable Last seen, please contact your Knox Guard administrator.
    SERVICE ENDS Date after which the device is no longer supported by Knox Guard under the terms of its license.
    CARRIER Device carriers that support the device.
    ANDROID VERSION Current Android version of the device.
    AGENT VERSION Current agent version installed on the device.
    UPLOADED Upload status of the device. On uploaded devices, lists the date when the device was uploaded.
    ACCEPTED Indicates whether the device is accepted.
    ACTIVATED Indicates whether the device is activated.
    MODIFIED Date when a device action was last performed (for example, Update offline lock).
    PREVIOUS IMEI/SNs Previous values the device was registered under.
    AUTO-LOCK Indicates whether auto-lock is enabled for the device.
    SIM POLICY Current policies applied to the device.
    WIPEOUT Indicates whether the device has been successfully wiped.
    BLOCK FACTORY RESET Indicates whether factory reset is blocked for the device.
    OFFLINE LOCK Indicates whether the offline lock policy is enabled for the device.
    ENROLLMENT NOTICE Indicates whether an enrollment notice is enabled for the device.
    APP BLOCKLIST Indicates whether an app blocklist is enabled on the device.
    APP INSTALLATION

    Indicates whether the Application installation policy is enabled.

    This feature is available only by request. To enable Application installation, please contact your Knox Guard admin.
    ADB COMMAND Whether Android Debug Bridge (adb) communication is blocked on the device.
    DEVELOPER MODE Whether access to the Developer options menu is blocked on the device.
    AE PROVISIONING Whether Android Enterprise provisioning is blocked on the device.
    FIRMWARE VERSION Current firmware version of the device.
    LATEST FIRMWARE Latest firmware version available for the device.
    FIRMWARE UPDATE CONTROL­­ Whether firmware update control settings are enabled for the device.
    FORCE WIFI AUTO DOWNLOAD­­ Whether automatic firmware updates over Wi-Fi are enabled for the device.
    WALLPAPER RESTRICTIONS­­ Whether wallpaper restrictions are applied to the device.

Download select devices in bulk

To download information for specific devices in bulk as a CSV file:

  1. Click ACTIONS > Search device(s).
  2. Upload a CSV file with the IMEIs or serial numbers of the devices you want to search for. A maximum of 10,000 devices can be queried at a time.
  3. Click SUBMIT.
