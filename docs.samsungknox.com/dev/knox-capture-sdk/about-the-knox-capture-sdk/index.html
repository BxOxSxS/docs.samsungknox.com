About the Knox Capture SDK

The Knox Capture SDK helps mobile application developers turn their enterprise Android smartphones or tablets into powerful barcode scanners.

With simple-to-use APIs, developers can access device cameras to read 1D and 2D barcode symbologies, display important scanning information in the camera UI, and push captured data to a user’s business productivity app or web browser.

Requirements

Before you begin with app development using the Knox Capture SDK, you’ll need to ensure that you have the following:

  • Android Studio IDE
  • Compatible Android mobile devices secured by Samsung Knox

Unsupported devices

The following Samsung devices are not supported:

  • A series — A01, A02s, A03Core, A04s, A11, A13, A13-5G
  • F series — F02s, F13
  • M series — M02s, M01, M11, M13, M13-5G

Availability

The availability of Knox Capture SDK may vary by country or region. Please check our list of supported Knox Capture locations for more information.

Supported barcode symbologies

The Knox Capture SDK currently allows developers to create scanning apps that can read the following barcode symbologies:

1D barcodes (alphanumeric and numeric)

  • Codabar
  • EAN13
  • EAN8
  • Interleaved 2 of 5
  • UPCA
  • UPCE
  • Code 128
  • Code 39
  • Code 93

2D barcodes

  • Data Matrix
  • PDF417
  • QR
