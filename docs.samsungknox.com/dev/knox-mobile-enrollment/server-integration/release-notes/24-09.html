Knox Mobile Enrollment Server Integration 24.09 release notes

Last updated September 25th, 2024

Enhanced authentication with OAuth 2.0

Previously, Knox Mobile Enrollment only supported Knox Cloud Authentication to provide secure access to APIs. Knox Mobile Enrollment 24.09 now supports two authentication schemes to access and use APIs:

  • existing Knox Cloud Authentication

  • new Knox OAuth 2.0 Authentication

In OAuth 2.0, scopes are used to control granular permissions for API access granted to an app. When you define a scope for Knox Mobile Enrollment’s integration with an app:

  1. The corresponding terms and conditions are showed to the device user for consent.

  2. After the device user accepts the terms and conditions, scopes are used to control which APIs have access to the device. All requests outside the defined scope are rejected.

For information about the OAuth 2.0 scopes available for Knox Mobile Enrollment, see Scopes for Knox cloud services APIs.

Learn more about admin features

To learn about features available to admins in this release, see Knox Mobile Enrollment 24.09 release notes in the admin guide.
