Samsung India Identity SDK FAQs

General

The Samsung India Identity SDK is an application programming interface(API) for app developers to develop apps which meet the Aadhaar compliant authentication according to the Aadhaar Authentication API Specification. Apps developed using the Samsung India Identity SDK APIs can obtain the user authentication, or get the e-KYC data based on the Aadhaar identity from the Aadhaar server.

Aadhaar is a 12-digit unique identification (UID) number given to residents of India. This Aadhaar is a biometric database that includes an iris scan data, digital fingerprints, a digital photo, and text-based data for every resident. Aadhaar is brand name of Unique Identification Authority of India (UIDAI). UIDAI sets the technology standards and guidelines for devices which needs to be certified.

Samsung India Identity SDK is supported only on IR enabled Samsung devices running Android L. Currently, SM-T116IR supports Samsung India Identity SDK.

UIDAI has sent letters to existing AUA/KUAs to comply with following:

  • Vendor SDK to be level 0/1 certified by end of May, 2017.
  • All existing biometric devices to be level 0/1 registered by end of May, 2017.
  • UIDAI will discontinue all the public devices starting mid-July, 2017.

The following table depicts the comparison between Public and Registered Devices:

Feature Public Device Registered Device
Encryption Signing of Biometric not mandated in device SDK. Signing of Biometric is mandated in device. Level 0 - In SDK, Level 1 - In Trust Zone
Security Less Secure More Secure
Device Identification No Yes

Installation

Apps developed using the Samsung India Identity SDK APIs are supported on the enterprise-managed Knox devices (with IR capability). You can install your apps either in the user space, or in the Knox Workspace as required.

Licensing

You require a Samsung India Identity license key to use the SDK. You can request for a development, or for a production license key.

License types

  • Development – While you are still developing your app, you use this license key. A development key expires after 6 months, at which point you must get a new development license if you still need one.
  • Production – When you are ready to release your app commercially, you use this license key.

See also:

  • Get a license
  • Activate a license

You can activate a license with the activateLicense (String licenseKey, String pkgName) API method. Once you call this API method, a broadcast intent ACTION_LICENSE_STATUSis sent with the result of license activation or validation result.

Operation

If the device does not support the Samsung India Identity APIs, calling the APIs would throw a “Runtime Exception”. If there is no runtime exception thrown, then the device supports the Samsung India Identity SDK.

Using the Samsung India Identity SDK, you can capture the iris image for either single or both eyes. You can call startActivityForResult(intent,requestCode) or capture(String pidOptions) to capture the iris image. In case of a single eye, set iCount value as “1”, and for dual eyes, set as iCount as “2” in the PidOptions XML.

By default, all devices released in market will get registered only with UIDAI production server. For testing authentication/e-KYC with production server, you will need to tie up with an AUA/ASA service provider. During the development phase, you can use the UIDAI staging server for testing authentication/e-KYC. In this case, the device should be registered with UIDAI staging server instead of the production server. You need to contact Samsung support and share the IMEI number of devices which has to be tested with staging server. Once the IMEI numbers are updated, then you can re-initiate the registration flow by clearing the India identity application data as mentioned in this guide.

You must ensure that your enterprise-managed devices behind firewall have access to the following servers:

Samsung license servers — so that when you activate Samsung India Identity SDK license, devices can verify their license keys. Devices periodically check their licenses a few times a week.

  • gslb.secb2b.com:443
  • eu-elm.secb2b.com:443
  • eu-prod-klm.secb2b.com:443

Samsung management server — is managed by the Samsung via which security policies related to key rotation can be enforced upon the devices. In addition, it also registers/deregisters the devices with UIDAI registered device server.

  • rdserv.mfisiel.com:443

If your device is behind a firewall, please contact support to get the complete list of URLs to be allowed access.
