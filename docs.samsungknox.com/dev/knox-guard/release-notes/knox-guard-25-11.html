Knox Guard 25.11 release notes

Last updated November 6th, 2025

This document is new for the Knox cloud services 25.11 UAT.

On this tab

  • API support for overdue payment wallpaper policy
  • Improved error handling when uploading devices
  • Get Device Info API enhancements

API support for overdue payment wallpaper policy

To encourage users to fulfill outstanding payments, Knox Guard 25.11 adds new APIs that allow you to apply wallpaper restrictions to devices. If you have a Knox Guard Advanced license, you can now display an overdue payment wallpaper on a device’s home and lock screens. Users can’t edit this wallpaper until restrictions are removed.

To enable this feature on a specific device, use the Set wallpaper API to upload a wallpaper, then use the Update wallpaper restrictions policy API to apply the restriction. You can also use the corresponding Async API to apply overdue payment wallpapers to multiple devices in bulk.

Improved error handling when uploading devices

Knox Guard 25.11 improves error handling when using the Upload Devices API. If you set applySimControl to true but haven’t set a default SIM control policy, the Upload Devices endpoint now returns a 412: Precondition Failed error.

To resolve this error, you can use the Set SIM control policy API to create a default SIM control configuration, or ensure applySimControl is set to false when using the Upload Devices API.

Get Device Info API enhancements

With Knox Guard 25.11, the Get Device info API endpoint now returns serial number, device model, and device Android operating system information.
