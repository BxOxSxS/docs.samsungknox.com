Knox Guard 26.06 release notes

Last updated June 4th, 2026

This document is new for the Knox cloud services 26.06 UAT.

On this tab

  • New
  • API support for application installation policy
  • Blinking relock reminders
  • Updates
  • Get Device Info now returns full device SKU

New

API support for application installation policy

Starting with Knox Guard 26.06, you can use new endpoints to configure the application installation policy.

Endpoint Use to:
Get application installation policy Retrieve the current application installation policy for your tenant.
Upload new app to application installation policy

Add a new app to install on devices. Only one app can be installed at a time.

If you have a Knox Guard Advanced license, you can install up to a maximum of 5 apps at a time.
Update application installation policy Update the app list of the application installation policy.
Delete apps in the application installation policy Remove apps from the application installation policy's app list.

Blinking relock reminders

Previously, pay-as-you-go (PAYG) tenants could only configure one-time relock reminders. With Knox Guard 26.06, you can format relock reminders as blinking messages and set how often they display on devices.

The Set Relock reminders request body has a new blinkReminder field that lets you display relock reminders as blinking messages, specify the duration between reminders, and specify when blinking reminders can be shown to users.

Updates

Get Device Info now returns full device SKU

With Knox Guard 26.06, the Get Device info endpoint response features a new fullModel field listing the device’s complete SKU number. You can use this information to identify the device’s specifications.

See Knox Guard API reference for more information.
