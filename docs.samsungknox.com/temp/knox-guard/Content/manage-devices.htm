Manage devices

On this page

General Device Details

Use the Knox Guard Devices screen to review the payment compliance and payment history of specific devices. Individual devices display as a link that can be selected when needed to review specific devices in greater detail.

Based on device compliance status and payment history, the ACTIONS drop-down menu can be invoked to accept or reject devices, start and stop overdue payment notifications, send device payment reminders, lock and unlock devices and complete subscription payments.

To review the status of Knox Guard managed devices:

  1. Select Devices from the left-hand navigation menu. Devices can be also located using a device's IMEI or approval ID within the Search function located on the top, left-hand side, of the screen. The approval ID search string can include these special characters (@, :, $, /, *, %, ^, &, \, (, ), +, ?, {}, [],).

A maximum of seven columns can be displayed within the Devices screen. Select columns by invoking the drop-down menu from the ellipsis (...) on the far right of the screen.

Display options include IMEI/SN, Model, Status, Approval ID, Service ends, Carrier, Android version, and Modified.

NOTE - An App version display item is also available to devices with apps registered (uploaded) from the KG console. App version can be selected when App Installation is available for the listed tenant. However, the App version parameter is gated behind a feature flag and is only enabled by request. Please contact your Knox Guard administrator if wanting to enable the App version parameter
NOTE - There is also a Last seen option that displays when a listed device was last detected by the Knox Guard server. However, the Last seen parameter is gated behind a feature flag and is only enabled by request. Please contact your Knox Guard administrator if wanting to enable the Last seen parameter.
  1. The IMEI/SN column lists device IDs used as device identifiers within the Knox Guard console. The IMEI can also be used to read devices into the Knox Guard console using a properly formatted CSV file with one device IMEI per line. Additionally, the IMEI/SN column can be filtered to optionally display SIM control enabled devices and listed and unlisted SIM changes to help administrator how many users have attempted to swap their SIM. A SIM card icon indicates when a SIM card swap is detected. A pop-up displays when hovering over the icon describing the SIM card swap attempt in greater detail.

NOTE - Each device's IMEI/SN appears as a link that can be selected to display device information in greater detail. The Device details screen lists the selected device's IMEI/SN, Model, Status, Carrier, Android version, SIM control, Approval ID (billing approval ID), subscription Paid count, payment Total count, Licenses (displayed as a link to invoke license details), Service start date, Service end date, who the device was Uploaded by, when the device was Last updated, and the date the device Added into Knox Guard. Use the < Previous | Next > navigation functions to scroll either forwards or backwards through different Device details screens as listed sequentially within the Devices screen.

NOTE - If the Device details screen displays a Status of Activating, Locking, Unlocking, Completing, Starting Reminder, or Stopping Reminder a Cancel link displays for stopping the listed in-process operation. Additionally, an App version item displays for devices with apps registered (uploaded) from the KG console. The bottom of each Device details screen also displays a View Device Log link that can be selected to review the device's transaction action, and resulting status changes and transaction time. For more information, go to: Device Logs.
  1. Refer to the MODEL column to review the list of models populating the Knox Guard console. Optionally use the column's filter function to refine which device models display.
  2. Refer to the STATUS column to assess whether each listed device is Activating, Active, Completing, Locked, Locking, Payment Complete, Pending, Rejected, has a Reminder On, Resetting, Starting Reminder, Stopping Reminder, Unlocking or Exchanging.
NOTE - To review a device's Knox Guard status flow once accepted into the console, go to: Knox Guard status flow.
NOTE - When payment information is loaded into Knox Guard, an overdue device can update to Payment complete | X days since loan was repaid, with X equal to the actual number of days since the loan was repaid and the Device status refreshed. Similarly, when payment information is uploaded, a device could update to Overdue | X days in arrears, with X equal to the number of days payment is delinquent.
NOTE - If the Device status reflects Locked, Unlocking, Reminder ON or Stopping Reminder, a PIN button appears to right of the status as an indicator that a PIN must be input on the user's device to stop reminders or unlock the device.
  1. Refer to the APPROVAL ID column to review the ID assigned when the device was approved for Knox Guard management.
  2. The SERVICE ENDS column lists the date when the listed device is no longer supported by Knox Guard under the terms of its license.
  3. Refer to the CARRIER column to review and filter or search for various device carriers supporting each listed device.
  4. Refer to the ANDROID VERSION column to assess whether each listed device is using a current or legacy version of Android.
  5. Refer to the MODIFIED column to assess when the status of each listed device IMEI/SN was refreshed.

Device Log

The Knox Guard device log auto-refreshes as changes or updates are made to a selected device. To view the device log, refer to either a or b:

  1. Click the IMEI/SN link from within the Devices screen to open the Device details pop-up, and select the View device log link near the bottom of the Device details screen.
  2. Select the checkbox next to a device and select View device log from the ACTIONS drop-down menu on the right hand side of the Devices screen.

The Knox Guard device log displays the following:

  • ACTION - Lists the selected device's enrollment activity, notifications and blinking reminders, device lock/unlock operations and subscription payments.
  • DEVICE STATUS - Displays the device's status updates as listed actions are initiated or completed. For more information, go to: Knox Guard status states.
  • DETAILS - Describes each listed action's enrollment activity, device PIN usage and service updates.
  • MODIFIED - Lists each action's occurrence time to provide a chronology of enrollment, payment and service activities.

Select EXPORT TO CSV to archive the device log in CSV file format to a location of your choosing.

Select OK to close the Device Log and return to the Device details screen.
