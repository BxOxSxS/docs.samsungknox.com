Overdue payment notifications and reminders

On this page

Knox Guard devices that are overdue, but not necessarily delinquent, in their payment status can be sent both reminder notifications and escalated blinking reminders directly to the user device. Notification options can be customized with escalating severity as needed.

Send overdue notifications

To send customizable overdue notifications as payment reminders to specific devices:

  1. Select specific overdue devices from the left-hand side of the Devices screen.
  2. Navigate to the ACTIONS drop-down menu from the top, right-hand side, of the Devices screen and select Send payment overdue notification.

  1. Select the Initially show notifications as a full-screen message on the device(s) to make notifications display as full screen message before the message displays on the notification panel. Selecting this option makes notifications difficult to ignore. Optionally select the Preview full-screen message link to preview how the full-screen message displays on the user device.
  2. Enter the Phone number of the contact resource managing the selected device(s) subscription payment. Special characters can be used if necessary.
  3. Refer to Notification 1, Notification 2, or Notification 3 and enter customized text for each notification with escalating notification urgency and severity as needed for each notification. If needed, include a properly formatted URL ( http://www...). When the device user taps the URL hyperlink, it opens a relevant Web page within the device browser. The URL is optional, but could be of assistance to subscribers who want to make device payment queries as seamless as possible. There is a 500 character maximum per notification. Select the checkbox of the notification best suited for this current payment notification.
  4. Select SEND to proceed with the overdue payment notification. A notification message displays on the top portion of the Devices screen stating the message has been sent. Additionally, an icon displays with device's overdue payment status within the Device status column.

Send overdue notifications in bulk

A standardized CSV template is available to read devices into the Knox Guard console using their IMEI for sending overdue payment notifications in bulk. The maximum number of devices that can be bulk notified is 10,000.

To bulk notify devices of their overdue payment status:

  1. Select BULK ACTIONS from the lower, left-hand, side of the screen.
  2. Select the Download CSV template link from the top portion of the screen.
NOTE - As an additional bulk notification option, an Excel file (.xls or .xlsx) can an be used if not wanting to use a CSV formatted file.
NOTE - If unsure of how to create a properly formatted CSV file for bulk overdue notifications, select Instructions for all bulk actions from the bottom, left-hand, side of the Bulk actions screen.
  1. Select the SEND PAYMENT OVERDUE NOTIFICATION portion of the Bulk actions screen and add the CSV file into the Send payment overdue notification screen.
  2. Select the Initially show notifications as a full-screen message on the device(s) to make notifications display as full screen message before the message displays on the notification panel. Selecting this option makes notifications difficult to ignore. Optionally select the Preview full-screen message link to preview how the full-screen message displays on the user device.
  3. Enter the Phone number device users call for making a payment and stopping overdue notifications. Special characters can be used if necessary.
  4. If necessary, change the Notification 1, Notification 2, or Notification 3 message as needed for this particular overdue notification and its intended severity. If needed, include a properly formatted URL (http://www...). When the device user taps the URL hyperlink, it opens a relevant Web page within the device browser. The URL is optional, but could be of assistance to subscribers who want to make device payment queries as seamless as possible.
  5. Select SEND to proceed with the overdue payment notifications. A notification message displays on the top portion of the Devices screen stating the message has been sent to the devices within the file.

Send blink reminders

When an overdue payment notification fails to result in a prompt payment, a blinking payment reminder can also be sent to the device as an escalated payment reminder. The blink reminder can be stopped from the Knox Guard console or when a device user correctly enters the PIN on their device.

To start and stop blink reminders for selected devices:

  1. Select specific overdue device checkboxes from the left-hand side of the Devices screen to send blinking reminders.
  2. Invoke the ACTIONS drop-down menu from the top, right-hand side, of the Devices screen and select Start blinking reminder.

NOTE - The screen displays how the reminder message will appear on the recipient device.
 
  1. Set the periodic blink reminder Period from 2 seconds (minimum) to 24 hours (maximum). The default is period is 300 seconds.
  2. Enter a 200 character maximum Message device users receive about their delinquent subscription payment status and the blink reminders. If needed, include a properly formatted URL (http://www...). When the device user taps the URL hyperlink, it opens a relevant Web page within the device browser. The URL is optional, but could be of assistance to subscribers who want to make device payment queries as seamless as possible.
  3. Enter the Phone number of the contact managing the selected device(s) subscription payment. Either a phone number or E-mail address is required as a contact resource. Special characters can be used if necessary for the phone number.
  4. Enter the E-mail address of the contact managing the selected device(s) subscription payment. Either a phone number or E-mail address is required as a contact resource.
  5. Refer to the Set limits for when blinking reminder can be shown to users portion of the screen to define a period of time when blink reminders are not displayed to device users.
  • Do not show everyday - Select this option to define a specific time period for disabling blink reminders at the same time, each day of the week.
  • Do not show during the selected time period of - Select this option to enable fields where specific days and times can be set for disabling blink reminders.
  1. Select START to initiate the blink reminder for the specified interval. A notification message displays on the top portion of the Devices screen stating the blink reminder message has been sent to one more selected devices.
  2. Blink reminders can be stopped for selected devices by selecting Stop Blinking reminder from the ACTIONS menu.
NOTE - Blink reminders can also be stopped when the device user enters the PIN number on their device. The PIN can be accessed by selecting the PIN button on the Devices screen, within the Device status column next to the Reminder ON status indicator. Once the user enters the PIN, the blink reminder function is canceled. Blink reminders can also be stopped as a bulk device operation using a CSV, .xls or xlsx file.

  1. Upload a list of pre-existing devices in either CSV, XLS or XLSX format to use as the recipients of the blink reminders.
  2. Define the 200 character maximum Message devices receive informing them of their delinquent payment status. If needed, include a properly formatted URL ( http://www...). When the device user taps the URL hyperlink, it opens a relevant Web page within the device browser. The URL is optional, but could be of assistance to subscribers who want to make device payment queries as seamless as possible.

  4. Enter the Phone number of the contact managing the subscription payment. Either a phone number or E-mail address is required as a contact resource. Special characters can be used if necessary for the phone number.
  5. Enter the E-mail address of the contact managing the subscription payment. Either a phone number or E-mail address is required as a contact resource.
  6. Refer to the Set limits for when blinking reminder can be shown to users portion of the screen to define a period of time when blink reminders are not displayed to device users.
  • Do not show everyday - Select this option to define a specific time period for disabling blink reminders at the same time, each day of the week.
  • Do not show during the selected time period of - Select this option to enable fields where specific days and times can be set for disabling blink reminders.
  1. Select START to initiate the blink reminder for the specified interval. A notification message displays on the top portion of the Devices screen stating the blink reminder message has been started for the devices.
  2. Blink reminders can be stopped for selected devices by selecting Stop blinking reminder from the Devices screen ACTIONS menu or from the Bulk actions screen.
NOTE - Blink reminders can also be stopped when the user enters the PIN number on their device. Obtain the PIN by selecting the PIN button next to the Reminder ON device status indicator. Blink reminders are canceled once the user enters the PIN. Blink reminders can also be stopped as a bulk device operation using a CSV, .xls, or .xlsx file.
