Class Overview

This class provides to set device settings configuration


Specific Policy
  • KEY_NAVIBAR_BUTTON_ORDER change navibar's button order
  • KEY_TOUCH_SENSITIVITY enable touch sensitivity option in Settings
  • KEY_HWKEY set custom action for hardware key event
  • KEY_BRIGHTNESS sets the screen brightness level
  • KEY_SYSTEM_LOCALE This method sets the default language and region for the device
  • KEY_INPUT_METHOD set default device's keyboard
  • KEY_POWER_SAVING_MODE turns on/off the power saving option of the device

public static final String KEY_HWKEY

Since: API level 34

Deprecated in API level 37

API to set custom action for hardware key event.

Non-Device Owner
App/Activity Launch
Short Press Long Press Double Press
Xcover/Active Key Supported Supported N/A
Top Key Supported Supported N/A
Hook Key N/A N/A N/A
Home Key N/A N/A N/A
Back Key N/A N/A N/A
Recent Key N/A N/A N/A
Volume Up Key N/A N/A N/A
Volume Down Key N/A N/A N/A
Side Key N/A N/A Supported


    Param
    PARAM_HWKEY_CODE The key code which are defined in KeyEvent.XXX
    PARAM_HWKEY_ACTION Target action when there is event. eg, "Activity" for launch tar get activity
    PARAM_HWKEY_ACTION_ACTIVITY The value for "Action"
    PARAM_HWKEY_ACTION_COMPONENTNAME component name for startActivity
    PARAM_HWKEY_ACTIVITY_EVENT Target event for startActivity
    PARAM_HWKEY_ACTIVITY_EVENT_SHORT_PRESS The value for "Event"
    PARAM_HWKEY_ACTIVITY_EVENT_LONG_PRESS The value for "Event"
    PARAM_HWKEY_ACTIVITY_EVENT_DOUBLE_PRESS The value for "Event"

    NOTE: The following params are temporarily not supported
    PARAM_HWKEY_ACTION_BROADCAST The value for "Action"
    PARAM_HWKEY_ACTION_INTENT custom intent for send broadcast
    PARAM_HWKEY_BROADCAST_DISPATCH To set disable original key event action. eg, "false" for disable original action, otherwise "true"
    PARAM_HWKEY_BROADCAST_TARGET_PACKAGE Target package name for send broadcast. eg, when it's not set, send implicit broadcast
    PARAM_HWKEY_BROADCAST_DEVICE_WIDE send boardcast to all users(include profile), when application using custom intent, device wide set to "false" automatically.
    PARAM_HWKEY_BROADCAST_DEFAULT_INTENT_ACTION If set broadcast intent as null, default broadcast intent will be set
    PARAM_HWKEY_BROADCAST_INTENT_EXTRA_REPORT_TYPE_NEW Broadcast intent extra regarding report type. It can be PARAM_HWKEY_BROADCAST_INTENT_EXTRA_KEY_ACTION_DOWN or PARAM_HWKEY_BROADCAST_INTENT_EXTRA_KEY_ACTION_UP
    PARAM_HWKEY_BROADCAST_INTENT_EXTRA_KEY_CODE Broadcast intent extra regarding key code.

Usage
 Uri uri = Uri.parse("content://com.samsung.android.knox.sdk/config/settings");
 ContentValues contentValues = new ContentValues();
 contentValues.put("component-name", "com.android.settings/com.android.settings.Settings");
 contentValues.put("event", "press");
 contentValues.put("key-code", "1015");
 contentValues.put("action", "activity");
 contentValues.put("gray-out", "false");
 String selection = "key=?";
 String[] selectionArgs = new String[] {"hardware-key"};
 mContext.getContentResolver().update(uri, contentValues, selection, selectionArgs);

 Uri uri = Uri.parse("content://com.samsung.android.knox.sdk/config/settings");
 String selection = "key=? AND key-code=? AND action=? AND event=?";
 String[] selectionArgs = new String[] {"hardware-key", "24", "activity", "long-press"};
 mContext.getContentResolver().delete(uri, selection, selectionArgs);
